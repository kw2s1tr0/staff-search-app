# AI Working Rules

## 基本方針
- 作業前に `AGENTS.md` を確認し、今回のユーザー指示と矛盾しない範囲で従う。
- 既存コード、設定、命名、ディレクトリ構成を先に確認してから変更する。
- 変更は必要最小限にし、無関係なリファクタリングや整形を混ぜない。
- 推測を確定情報として扱わず、不明点は不明と明記する。

## 指示の優先順位
指示や文書が競合する場合は、原則として以下を優先する。

1. 今回のユーザーの明示指示
2. より近い階層にある `AGENTS.md`
3. ルートの `AGENTS.md`
4. `docs/instructions/`
5. その他のドキュメント

## 既存差分の扱い
- 作業前に可能な範囲で `git status` を確認する。
- ユーザーや他者が作った変更を勝手に戻さない。
- 無関係なファイルの整形、削除、移動、修正をしない。
- 既存差分が作業対象に関係する場合は、内容を読んで共存する形で進める。

## docs の扱い
- `docs/` 配下に該当文書がない場合でも、必要がなければ新規作成しない。
- `docs/workbench/` への記録は、複雑な調査、判断分岐、未解決事項、後で参照すべき理由がある場合に限る。
- 軽微で完結した修正では、不要な作業メモを増やさない。
- 再利用できる知識は `docs/skills/`、今回限りの判断メモは `docs/workbench/` に分ける。

## 機密情報
- 機密情報や認証情報を文書、ログ、コメント、作業メモに記録しない。
- `.env`、ログ、DB ダンプ、スクリーンショットに秘密値が含まれる可能性を前提に扱う。
- 外部共有や恒久文書に秘密値、トークン、パスワード、個人情報を転記しない。
